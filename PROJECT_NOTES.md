# TG Poster Notes

## Текущее состояние

Проект представляет собой desktop-приложение на Python для автопостинга в Telegram.

Основные компоненты:
- `main.py` — точка входа.
- `ui.py` — графический интерфейс на `customtkinter`.
- `telegram_service.py` — фоновый Telegram-клиент и планировщик публикаций.
- `repository.py` — работа с SQLite и медиафайлами.
- `config.py` — пути и загрузка `.env`.

## Что уже сделано

### Запуск на Windows

Проблема:
- `start.ps1` блокировался Windows / PowerShell Execution Policy.

Что добавлено:
- `start.bat` — запуск без PowerShell.
- `build_exe.bat` — сборка `exe` через `PyInstaller`.

Практический вывод:
- Для обычного локального запуска удобно использовать `start.bat`.
- Для standalone-запуска используется `tg_poster.exe`.

### Поддержка exe

В `config.py` добавена логика определения рабочей директории через `sys.executable`, если приложение запущено как собранный `exe`.

Это нужно для того, чтобы рядом с `exe` корректно подхватывались:
- `.env`
- `database.db`
- `media`
- `app.log`
- `my_account.session`

Итог:
- `tg_poster.exe` можно положить в корень проекта рядом с этими файлами, и приложение работает корректно.

### Русификация интерфейса

В `ui.py` переведены на русский:
- заголовки
- вкладки
- кнопки
- подписи полей
- сообщения в `messagebox`

В `telegram_service.py` переведены:
- статусы подключения
- сообщения логов
- сообщения о публикации

Примечание:
- после любой правки интерфейса нужно пересобирать `exe`, иначе старый `exe` останется со старыми строками.

### Вставка текста в поля

Проблема:
- вставка через `Ctrl+V` работала только на английской раскладке;
- через правую кнопку мыши работала корректно.

Что сделано:
- в `ui.py` добавлены:
  - `Ctrl+C`
  - `Ctrl+V`
  - `Ctrl+X`
  - `Ctrl+A`
  - контекстное меню по ПКМ
- добавлена обработка по `keycode`, чтобы горячие клавиши работали независимо от раскладки клавиатуры.

## Как сейчас запускать проект

### Вариант 1. Python

Запуск:
- `start.bat`

Или напрямую:
- `venv\Scripts\python.exe main.py`

### Вариант 2. EXE

Сборка:
- `build_exe.bat`

После сборки:
- готовый файл лежит в `dist\tg_poster.exe`
- для использования его можно положить в корень проекта рядом с рабочими файлами

На практике уже проверено:
- `tg_poster.exe`, перенесенный из `dist` в корень проекта, подхватывает данные и запускается.

## Как работает расписание

Кнопка `Перезагрузить расписание`:
- вручную пересобирает jobs планировщика из базы данных;
- удаляет старые задачи;
- создает новые задачи по активным объявлениям и времени публикации.

Важно:
- после сохранения объявления или цели перезагрузка расписания уже вызывается автоматически;
- кнопка нужна как ручной принудительный refresh.

## Важное ограничение текущей архитектуры

Чтобы автопостинг работал, приложение сейчас должно быть запущено.

Причина:
- планировщик и Telegram-клиент работают внутри процесса приложения;
- если закрыть окно и завершить процесс, публикации прекращаются.

Сейчас проект не является:
- Windows service
- фоновым daemon-процессом без UI
- облачным сервисом

## Если нужен постинг без открытого окна

Варианты:
- сделать сворачивание в трей;
- сделать headless-режим без GUI;
- вынести на VPS / сервер;
- сделать автозапуск вместе с Windows.

## Если нужен постинг независимо от этого ПК

Когда компьютер выключен, локальное приложение не может публиковать.

Для полной независимости нужен сервер:
- VPS
- удаленный Windows/Linux-сервер
- домашний мини-ПК, работающий 24/7

## Что обсуждали по VPS

### Oracle Free Tier

Oracle Cloud Free Tier подходит лучше большинства бесплатных вариантов.

Подтверждено по официальным источникам:
- есть `Always Free`, а не только trial;
- доступны бесплатные `AMD-based Compute` и `Ampere A1 Compute`;
- для ARM доступно больше ресурсов, чем для AMD micro;
- free-ресурсы могут быть недоступны из-за capacity;
- для регистрации обычно нужна банковская карта для верификации.

Источники:
- https://www.oracle.com/cloud/free
- https://docs.oracle.com/en-us/iaas/Content/FreeTier/freetier_topic-Always_Free_Resources.htm
- https://www.oracle.com/cloud/compute/arm/pricing/

### Что лучше для этого проекта

Рекомендация:
- лучший бесплатный вариант: `Oracle ARM (Ampere A1)`
- запасной бесплатный вариант: `Oracle AMD Free`
- лучший вариант по надежности: дешевый платный VPS

### Минимальные ресурсы для headless-версии

Минимум:
- `1 vCPU`
- `512 MB RAM`
- `10-20 GB SSD`

Нормально:
- `1 vCPU`
- `1 GB RAM`
- `20 GB SSD`

С запасом:
- `2 vCPU`
- `2 GB RAM`

Если одновременно запущено несколько экземпляров приложения, `Telethon` может блокировать `my_account.session`.

Сейчас в `telegram_service.py` добавлена более понятная диагностика:
- вместо сырого `database is locked`
- показывается, что `сессия Telegram занята другим экземпляром приложения`

### Поддержка тем в группах Telegram

Добавлена поддержка публикации в конкретные темы forum-group.

Теперь у канала можно хранить:
- `topic_id`
- `topic_title`

Это сделано в:
- `repository.py`
- `telegram_service.py`
- `web_app.py`
- Vue-интерфейсе

### Как это работает

Для каналов с темами:
1. В карточке канала указывается `chat_ref`
2. Нажимается `Загрузить темы`
3. Приложение получает темы через текущий Telegram-аккаунт
4. Выбирается нужная тема
5. При публикации сообщение уходит именно в выбранную тему
