# TG Poster

Веб-приложение на Python и Vue для автопостинга объявлений в Telegram.

Проект позволяет:
- хранить объявления, чаты и расписание в `SQLite`;
- публиковать текст и медиа в один или несколько Telegram-чатов;
- запускать ручную публикацию;
- смотреть логи публикаций в браузере.

## Стек

- Python
- `customtkinter`
- `Telethon`
- `APScheduler`
- `FastAPI`
- `Jinja2`
- `Vue 3`
- `Vite`
- `SQLite`

## Структура проекта

- `main.py` - точка входа web/API
- `worker_main.py` - отдельный процесс worker для Telegram и расписания
- `web_app.py` - веб-приложение и маршруты
- `frontend/` - Vue-интерфейс
- `telegram_service.py` - Telegram-клиент и планировщик
- `repository.py` - работа с базой данных и медиафайлами
- `config.py` - загрузка настроек и путей
- `logger_config.py` - настройка логирования
- `templates/` - legacy HTML-шаблоны
- `static/` - legacy стили
- `start.bat` - запуск web/API на Windows
- `start_worker.bat` - запуск worker на Windows

## Возможности

- создание и редактирование объявлений;
- привязка одного объявления к нескольким чатам;
- несколько времен публикации для одного объявления;
- настройка интервала в днях;
- включение и отключение объявлений и целей;
- публикация текста без медиа или с одним/несколькими файлами;
- кнопка `Опубликовать сейчас`;
- ручная кнопка `Перезагрузить расписание`;
- Vue-интерфейс для объявлений, целей и логов;
- загрузка медиафайлов через браузер.

## Настройка Telegram

1. Получите `api_id` и `api_hash` на `https://my.telegram.org`.
2. Создайте файл `.env` рядом с `main.py`.
3. Заполните его по примеру из `.env.example`.

Пример:

```env
TG_API_ID=12345678
TG_API_HASH=your_api_hash
TG_PHONE=+77001234567
TG_PASSWORD=
WEB_HOST=127.0.0.1
WEB_PORT=8000
EMBEDDED_WORKER_ENABLED=0
```

Примечания:
- `TG_PASSWORD` нужен только если у аккаунта включена двухфакторная защита.
- `WEB_HOST` и `WEB_PORT` задают адрес локального веб-сервера;
- `EMBEDDED_WORKER_ENABLED=0` означает, что worker запускается отдельным процессом;
- при первом запуске `Telethon` может запросить код подтверждения или пароль;
- Telegram-сессия сохраняется локально в `my_account.session`.

## Запуск из исходников

Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv venv
venv\Scripts\python.exe -m pip install --upgrade pip
venv\Scripts\python.exe -m pip install -r requirements.txt
cd frontend
npm install
cd ..
```

Запуск web:

```powershell
start.bat
```

Запуск worker (в отдельной консоли):

```powershell
start_worker.bat
```

или напрямую:

```powershell
venv\Scripts\python.exe main.py
venv\Scripts\python.exe worker_main.py
```

После запуска откройте в браузере:

```text
http://127.0.0.1:8000/app
```

Legacy-версия на серверных шаблонах пока сохранена и доступна по:

```text
http://127.0.0.1:8000/ads
http://127.0.0.1:8000/targets
http://127.0.0.1:8000/logs
```

## Как это работает

1. `Vue` собирается в `frontend/dist`.
2. Приложение поднимает локальный `FastAPI`-сервер.
3. `FastAPI` отдает собранный Vue SPA по `/app`.
4. Backend API обслуживает объявления, цели, логи и действия планировщика.
5. Worker-процесс запускает Telegram-клиент и `APScheduler`.
6. Объявления, цели и расписание хранятся в `SQLite`.
7. Web сохраняет задачи публикации в очередь `publish_jobs`.
8. Worker обрабатывает очередь и отправляет сообщения в выбранные чаты.
9. Идемпотентность target-слотов контролируется таблицей `publish_dedup`.
10. Результаты записываются в лог и в таблицу `publish_log`.

## Ограничение текущей архитектуры

Автопостинг работает, пока запущен worker-процесс.

Сейчас проект не работает как:
- Windows service;
- фоновый daemon без интерфейса;
- облачный сервис.

Если остановить worker-процесс, новые публикации не будут выполняться, но web продолжит ставить задачи в очередь.
