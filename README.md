# TG Poster

Desktop-приложение на Python для автопостинга объявлений в Telegram.

Проект позволяет:
- хранить объявления, чаты и расписание в `SQLite`;
- публиковать текст и медиа в один или несколько Telegram-чатов;
- запускать ручную публикацию;
- смотреть логи публикаций в интерфейсе;
- собирать standalone `exe` для Windows.

## Стек

- Python
- `customtkinter`
- `Telethon`
- `APScheduler`
- `SQLite`

## Структура проекта

- `main.py` - точка входа
- `ui.py` - графический интерфейс
- `telegram_service.py` - Telegram-клиент и планировщик
- `repository.py` - работа с базой данных и медиафайлами
- `config.py` - загрузка настроек и путей
- `logger_config.py` - настройка логирования
- `start.bat` - запуск проекта на Windows
- `build_exe.bat` - сборка `exe`

## Возможности

- создание и редактирование объявлений;
- привязка одного объявления к нескольким чатам;
- несколько времен публикации для одного объявления;
- настройка интервала в днях;
- включение и отключение объявлений и целей;
- публикация текста без медиа или с одним/несколькими файлами;
- кнопка `Опубликовать сейчас`;
- ручная кнопка `Перезагрузить расписание`;
- русифицированный интерфейс;
- горячие клавиши `Ctrl+C`, `Ctrl+V`, `Ctrl+X`, `Ctrl+A` независимо от раскладки.

## Требования

- Windows
- Python 3.11+ рекомендуется
- Telegram API credentials: `api_id` и `api_hash`

## Настройка Telegram

1. Получите `api_id` и `api_hash` на `https://my.telegram.org`.
2. Создайте файл `.env` рядом с `main.py`.
3. Заполните его по примеру из `.env.example`.

Пример:

```env
TG_API_ID=12345678
TG_API_HASH=your_api_hash
TG_PHONE=+77001234567
TG_PASSWORD=
```

Примечания:
- `TG_PASSWORD` нужен только если у аккаунта включена двухфакторная защита.
- при первом запуске `Telethon` может запросить код подтверждения или пароль;
- Telegram-сессия сохраняется локально в `my_account.session`.

## Запуск из исходников

Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv venv
venv\Scripts\python.exe -m pip install --upgrade pip
venv\Scripts\python.exe -m pip install customtkinter telethon apscheduler
```

Запуск:

```powershell
start.bat
```

или напрямую:

```powershell
venv\Scripts\python.exe main.py
```

## Сборка EXE

```powershell
build_exe.bat
```

После сборки файл появится в `dist\tg_poster.exe`.

Для корректной работы рядом с `exe` должны находиться рабочие данные проекта:
- `.env`
- `database.db`
- `media`
- `app.log`
- `my_account.session`

## Как это работает

1. Приложение запускает GUI.
2. В фоне стартует Telegram-клиент и `APScheduler`.
3. Объявления, цели и расписание хранятся в `SQLite`.
4. При наступлении времени публикации сообщение отправляется в выбранные чаты.
5. Результаты записываются в лог и в таблицу `publish_log`.

## Ограничение текущей архитектуры

Автопостинг работает только пока приложение запущено.

Сейчас проект не работает как:
- Windows service;
- фоновый daemon без интерфейса;
- облачный сервис.

Если закрыть приложение и завершить процесс, публикации остановятся.

## Что можно развивать дальше

- `headless`-режим без GUI;
- сворачивание в трей вместо полного закрытия;
- защита от двойной публикации;
- ретраи при ошибках Telegram;
- ротация логов;
- запуск на VPS или сервере 24/7.

## Безопасность

Не публикуйте в репозиторий:
- `.env`
- `my_account.session`
- `database.db`
- `media/`
- `app.log`

Эти файлы уже исключены через `.gitignore`.
